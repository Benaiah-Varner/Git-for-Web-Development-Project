 1. What is Semantic HTML? 
 Semantic HTML is html that gives context/meaning to the code.
    2. What is HTML used for? 
    Marking up a webpage and giving it its structure or skeleton.

    3. What is an attribute and where do we put it? 
    An attribute goes inside the opening tag of an element and it acts as a modify for that element. It also is used for specifity such as giving an element a class or an ID

    4. What is the h1 tag used for? How many times should I use it on a page?
    It acts as a headline or title of a page and should only be used once.

    5. Name two tags that have required attributes
    <img> has a required "src" attribute and <a> has a required "href" attribute.

    6. What do we put in the head of our HTML document? 
    The head is used for metadata or data about the HTML document. The head is usually placed between the <html> tag and the <body> tag.
    
    7. What is an id? 
    An ID is an attribute. An ID's value can only be used once in an HTML document.

    8. What elements can I add an id to? 
    All elements except <base> , <head> , <html> , <meta> , <script> , <style> , and <title>.

    9. How many times can I use the same id on a page?
    once

    10. What is a class? 
    A class is an attribute used for identification similiar to the ID attribute, excpet a class's value can be used on multiple elements throughout the HTML document. 

    11. What elements can I add a class to?
    Any element 

    12. How many times can I use the same class on a page? 
    As many times as needed

    19. What is the difference between a class and an id? - Give me an 
    example of when I might use each one.

    The main difference between the two is an ID's value can only be used once while a class's value can be used many times. Since an ID can only be used once it is mainly used for broader elements such as a section. On a webpage, you might use an ID in a section tag for the main content of the page. Since a class can be used for many different things it is more commonly used for smaller spectrum things such as a div or unordered list.

    37. Walk me through Lambda's git flow. 
    1. create your own version of repo
    2. Fork the repo and check to make sure your name  is in the corner and in the URL
    3. Add your TL as a collaborator on your forked repo.
    4. Clone your repo in GIT.
    5. Back inside the CLI, use the cd command to go ot the directory where you are going to want to clone your forked repo.
    6. Use the git clone command to clone the repo to the directory you are currently in.
    7. cd into the newly created repo usuing the cd <repo-name> command.
    8. Use the git checkout -b 'firstname-lastname' command to create your own branch name for the repo.
    9. Work on repo to start getting assigned project done.
    10. add commit changes to your branch, by using the git add command to get them ready to commit, then the git commit -m "followed by your message here" to commit them to the remote repository.
    11. push all commits to the remote repo by using git push -u origin branch-name command
    12. Now that the changes are on git hub you can submit a pull request
    13. Make sure it is your forked repo not the original.
